Anthropic が米政府の指令により Fable 5 と Mythos 5 へのアクセスを停止：国家安全保障とジェイルブレイク懸念が背景

2026年6月12日、米政府は国家安全保障当局を理由に輸出管理指令を発出し、Anthropic の最強力モデルである Fable 5 と Mythos 5 への外国籍（米国内外問わず）からのアクセスを停止するよう命じました。Anthropic は公式声明でこれを受け入れ、全顧客に対してこれらのモデルへのアクセスを即時無効にしたと発表しています。その他の Claude シリーズモデルへの影響はないと説明しています。

政府の指令は具体的な国家安全保障の懸念を開示していませんが、Anthropic はジェイルブレイク（プロンプトインジェクションやモデルの制約回避）のデモを確認したと指摘。そのデモでは既に知られている軽微な脆弱性が突かれたといい、同様の脆弱性は公開されている他のモデルでも同様に見つかると述べています。これにより、フロンティアモデルのセキュリティとガバナンスが改めて注目される形となりました。

この出来事は、AI モデルの力が増すにつれて国家レベルでの輸出管理やアクセス制限が現実化していることを示しています。今後はモデル提供者側でもセーフガードの透明性・監査可能性、そして政府との協力枠組みが求められるでしょう。

2026年6月のオープンウェイトモデルリリースウィーク：LLM・画像・音声・映像・3Dまで25+モデルが公開

2026年6月初旬、記録的な密度でオープンウェイトモデルが複数モダリティにわたってリリースされました。以下に主な発表をまとめます。\n\n## 大規模言語モデルとエッジチャット\n- Nemotron 3 Ultra（NVIDIA）：550B ハイブリッド Mamba–MoE、アクティブパラメータ 55B、コンテキスト 1M トークン、MMLU 89.1、NFP4 バリアントで Blackwell 上では約5倍のスループット。初の公開ウェイト 550B ハイブリッド Mamba–Transformer、データセンター エージェントスケール向け。\n- Gemma 4 12B（Google）：エンコーダーフリーの any-to-any（テキスト/画像/オーディオ/ビデオ）モデル、コンテキスト 256k、140+ 言語、AIME 2026 で 77.5、モバイル向け量子化トレーニング波（ONNX + MLX）。\n- LFM2.5-8B-A1B（Liquid AI）：エッジ向け MoE、アクティブ ~1.5B、コンテキスト 128k、MATH500 で 88.8、MLX レディ。オンデバイス数学・推論に強み。\n- Mellum2 Thinking（JetBrains）：最初のオープン JetBrains MoE、アクティブ 2.5B（64 エキスパート中 8）、コンテキスト 131k、LiveCodeBench v6 で 69.9、Apache 2.0。Qwen3-14B レベルのコーディング品質をアクティブ幅が小さい状態で実現。\n\n## 画像生成 — Ideogram 4 オープンウェイト\n- Ideogram 4 が初のオープンウェイトを公開。9.3B フローマッチング Diffusion Transformer (DiT)、ネイティブ 2K 解像度。結合ボックスとカラーパレットを含む構造化プロンプト(JSON)対応。ウェイトは Hugging Face (ideogram-ai/ideogram-4-nf4, FP8 バリアント) でゲート。ライセンスは Apache 2.0 コード；ウェイトは非商用契約（商用パスは Ideogram 経由）。関連リンク：技術ブログと Hugging Face コレクション。\n\n## 音声・音声・音楽 — 一週間に4つの TTS ラボ\n- Higgs Audio v3 TTS 4B（Boson AI）：100+ 言語、インライン感情/スタイル/プロソディタグ、歌/囁き/叫び、サブ秒タイム・トゥ・ファースト・オーディオ、8 コードブック AR デコーダー + 24 kHz 出力。\n- その他のラボ（ドットTTSなど）も音声合成・認識・音楽生成のオープンモデルをリリース。\n\n## ビジョン・ビデオ・3D\n- ビジョン VLMs と OCR、ビデオワールドと3Dシーン理解モデルもオープンウェイトで提供。\n\n## デプロイメント共通基盤\n- 多くのモデルは MLX、ONNX、vLLM などのランタイムへのエクスポートパスを提供し、エッジ・デバイスへの容易な導入を可能にしています。\n\nこの一週間のリリースは、オープンウェイトが言語だけでなくマルチモーダル全域にわたって加速していることを示しています。研究者・開発者はこれらをベースにカスタムファインチューンやドメイン特化アプリケーションを迅速に構築できるでしょう。
