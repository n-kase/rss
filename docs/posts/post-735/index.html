概要と事実

この記事は、四則演算を含む文字列をコンピュータで正しく計算する方法を、yacc（またはその互換ツールであるbison）を使って説明しています。まず、人間が式を解く際にかっこの優先順位や演算子の優先順位を意識していることを指摘し、コンピュータがそのままの式を処理すると「どこから計算を始めればよいか」がわからなくなる問題があることを示します。そこで、逆ポーランド記法（後置記法）に変換すれば演算の順序が明確になり、スタックというデータ構造を使って機械的に計算できることを説明します。スタックは数値を積み上げ、演算子が来たら上から二つ取り出して計算し、その結果をまた積むという単純なルールで動作します。次に、この変換と計算を自動で行うために文法を定義し、yaccに教える方法を示します。文法では演算子の優先順位を宣言し、再帰的な規則で「式とは式＋式であり、式×式であり、(式)であり、ただの数字でもある」と記述します。yaccはこの文法に従って入力式を解析し、各規則に付けたアクション（計算結果をスタックに積むなど）によって最終的な答えを得ます。さらに、エラー検出も文法定義だけで自動的に行われ、閉じかっこが足りない不正な式は構文解析時点でエラーとなります。最後に、計算そのものではなく逆ポーランド記法を出力するようにアクションを変更すれば、同じ文法定義で異なる出力も簡単に得られることを示しています。

背景と結論

コンピュータは元々「電子計算機」として計算を行うために生まれたため、四則演算のような基本的な計算処理はその根幹にあります。しかし、人間が自然に扱う中置記法は機械にとって解釈が難しく、優先順位やかっこの対応を明示するための中間表現が必要です。この記事では、その解決策としてスタックマシンと逆ポーランド記法、そして文法を駆使したパーサージェネレータ（yacc）の仕組みをわかりやすく紹介しています。これにより、プログラミング初心者でも「式を解く」ということがどのように機械的に実現できるかを具体的に理解でき、さらにエラー処理や出力形式の変更も柔軟に行えることがわかります。結論として、式の評価という一見単純な問題にも、言語処理系の基礎理論が深く関わっており、それを道具を使って実装することで複雑さを大幅に軽減できるということが示されています。

yacc に何をさせるかというと、まず文法を教えてあげます。「文法」とか言うと身構えちゃいますが、要は式ってこういうものだよという定義を書くだけです。

こんな感じ。

%left '+' '-'      /* 足し算・引き算はあとまわし（優先度ひくい） */
%left '*' '/'      /* 掛け算・割り算が優先（こっちが下=強い） */
%%
expr : expr '+' expr      /* 式とは「式 ＋ 式」である */
| expr '-' expr      /* または「式 − 式」である */
| expr '*' expr      /* または「式 × 式」である */
| expr '/' expr      /* または「式 ÷ 式」である */
| '(' expr ')'       /* または「( 式 )」である */
| NUM                /* または ただの数字である */
;

yacc では各ルールの後ろに { } で「アクション」(＝やってほしい処理)を書けます。

expr : expr '+' expr   { $$ = $1 + $3; }   /* 結果 = 左 + 右 */
| expr '-' expr   { $$ = $1 - $3; }
| expr '*' expr   { $$ = $1 * $3; }
| expr '/' expr   { $$ = $1 / $3; }
| '(' expr ')'    { $$ = $2;      }   /* カッコの中身がそのまま結果 */
| NUM             { $$ = $1;      }   /* 数字はその値そのもの */
;

$$このルール自身の計算結果

「左の式と右の式が出そろったら、足してそれを自分の値にして」
