概要と事実

MiniMaxは2026年6月12日に、総パラメータ数が約4,280億、アクティブパラメータが約23億のエキスパート混合構造（MoE）を採用したオープンウェイトの大規模言語モデル「MiniMax M3」のモデルウェイトをHugging Faceで公開しました。このモデルは文字・画像・動画といった複数の種類の入力を追加の変換器なしで直接処理できるマルチモーダル対応となっています。独自に開発した「MiniMax Sparse Attention」により、最大100万トークンまでの長い文脈を扱うことが可能となり、前世代と比べてプリフィル処理の速度が約9倍、デコード処理の速度が約15倍向上しています。ベンチマークテストでは、ソフトウェアエンジニアリングを評価するSWE‑Bench Proで59.0％を記録し、Gemini 3.1 Proの54.2％を上回りました。その他のエージェント連携やウェブ自律閲覧、OS操作などのテストでは、一部でGemini 3.1 Proに及ばないものもありますが、全体として互角の性能を示しています。ライセンスは非商用利用が無料で、年間売上が2,000万ドル未満の企業や個人は届け出と特定の表記のみで商用利用が認められます。モデルウェイトのサイズはbf16形式で約855GBですが、1-bit GGUFに圧縮すると約128GBまで小さくでき、比較的限られたハードウェアでも動作させやすいように設計されています。

ネットの反応と意見の変遷

公開直後、技術系の掲示板やSNSでは「オープンウェイトでここまで巨大なモデルが出てきた」という驚きの声が多く見られました。特に、専門家混合構造による計算効率の向上や、100万トークンという長い文脈窓に注目が集まり、今後の応用範囲の広がりに期待が寄せられました。一方で、モデルのサイズが大きいため実際に運用するには高性能なGPUが必要だという指摘や、ベンチマーク結果が特定のタスクでしか勝っていない点について「実務での優位性はまだ不明」という慎重な意見も出始めました。時間が経つにつれて、実際にモデルをダウンロードして試した開発者からは、量子化後の128GB版でも普通のワークステーションで動かせたという報告があり、ハードウェアのハードルが思ったより低いという肯定的な評価が増えてきました。また、ライセンス条件が比較的緩やかである点も、スタートアップや個人開発者にとって魅力的だと評価され、商用利用へのハードルが低いという声が強まっています。全体としては、初期の興奮とともに実用性への検証が進み、期待と現実のバランスを取る議論が活発になっています。

背景と結論

近年、大規模言語モデルのパラメータ数は急激に増加しており、同時にオープンソースやオープンウェイトでの公開が進んでいます。これにより、研究者や開発者は最先端の技術を自由に試すことができ、イノベーションのスピードが速まっています。MiniMax M3は、巨大な規模でありながら計算効率を高める独自の注意機構と、マルチモーダル入力への対応を備えることで、単なる言語処理だけでなく画像や動画を含む複合的なタスクにも適用しやすい点が特徴です。こうした動向は、AI技術が専門家だけでなく幅広いユーザーに届く基盤を築くものであり、今後はさらに軽量化や特化したタスク向けのファインチューニングが進むと予想されます。結論として、MiniMax M3はオープンウェイト大規模モデルの競争において一つの有力な選択肢となり、性能とアクセシビリティの両面から今後のAIエコシステムに影響を与える可能性が高いと言えます。

Gemini 3.1 Proと互角、4,280億パラメータLLM「MiniMax M3」公開

2026年6月13日 11:16

MiniMaxは6月12日、総パラメータ数が約4,280億でマルチモーダル対応のオープンウェイトLLM「MiniMax M3」のモデルウェイトをHugging Faceで公開した。アクティブパラメータ数が約230億のエキスパート混合モデル(MoE)アーキテクチャを採用する。ソフトウェアエンジニアリング評価「SWE-Bench Pro」ではGemini 3.1 Pro(54.2%)を上回る59.0%を記録した。

MiniMax M3は、総パラメータ数が約4,280億、アクティブパラメータが約230億のエキスパート混合モデル(MoE)アーキテクチャを採用するオープンウェイトLLM。入力はテキスト・画像・動画に対応し、いずれも追加のアダプタなしにネイティブで処理できる。同モデルは6月1日にAPIとして提供を開始していた。

アーキテクチャ上の最大の特徴は、独自開発の「MiniMax Sparse Attention(MSA)」だ。これにより最大100万トークンのコンテキストウィンドウを実現しつつ、前世代のMiniMax M2と比べてプリフィル速度を約9倍、デコード速度を約15倍に向上させた。1Mトークン処理時のトークンあたり計算量も20分の1に抑えられている。

コーディング系以外の評価でも、MCPサーバーとのエージェント連携を測る「MCP Atlas」では74.2%と、Gemini 3.1 Pro(69.2%)を上回った。一方、ウェブ自律閲覧の「BrowseComp」はMiniMax M3が83.5%でGemini 3.1 Proが85.9%。OS操作の「OSWorld-Verified」はMiniMax M3が75.2%(MiniMax公式Xアカウントの投稿では70.0%)でGemini 3.1 Proが76.2%など、すべての指標でGemini 3.1 Proを超えるわけではなく、全体として拮抗した水準にある。

ライセンスは非商用であれば無償での利用が可能だ。年間売上2,000万ドル未満の企業・個人による商用利用は、MiniMaxへの届け出と「Build with MiniMax」の表記のみで認められる。モデルウェイトのサイズはbf16フォーマットで約855GBだが、1-bit GGUFでは約128GBまで圧縮でき、より手頃なハードウェアでの動作を目指して意図的にパラメータ規模を抑えた設計になっている。
