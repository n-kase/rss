AIによる要約

中国のAI企業Moonshot AIが、コーディングに特化したオープンモデル「Kimi K2.7 Code」を公開した。このモデルは前世代よりもコーディング性能が大幅に向上し、同じ作業を行う際に必要な思考トークン数を約30％削減できるため、効率的にプログラムを生成できる。オープンソースとして誰でもダウンロードできるため、開発者や研究者が自由に利用・改良できる環境が整った。

この発表により、AIを活用したコーディング支援ツールの普及がさらに進むことが予想され、ソフトウェア開発のスピードや品質が向上する可能性がある。一方で、高性能なモデルが容易に手に入ることで、不正なコード生成やセキュリティリスクへの懸念も高まるかもしれない。ネット上では、技術の進化への期待とともに、使い方のルールや倫理的なガイドラインの必要性が議論される傾向が見られる。

したがって、開発者はこのような効率的なオープンモデルを積極的に活用しつつ、生成されたコードの品質や安全性をしっかりと確認する姿勢が求められる。また、提供側は利用者が誤用しないよう、適切な利用ポリシーや監視仕組みを整えることが重要である。

中国製AIモデル「Kimi K2.7 Code」がオープンモデルとして公開される、Kimiシリーズ最高のコーディング性能でトークン消費も少なめ

中国企業の** Moonshot AI**がAIモデル「

Kimi K2.7 Code」を2026年6月12日に公開しました。Kimi K2.7 Codeはコーディングエージェントに特化したAIモデルで、誰でもダウンロード可能なオープンモデルとして公開されています。

Kimi K2.7 Code: Open-Source Agentic Coding Model

https://www.kimi.com/resources/kimi-k2-7-code🌘 Kimi-K2.7-Code, our latest coding model, is now released and open-sourced!

— Kimi.ai (@Kimi_Moonshot)

🔷 Improved coding & agent performance over K2.6: +21.8% on Kimi Code Bench v2, +11.0% on Program Bench, and +31.5% on MLS Bench Lite.

🔷 Reasoning efficiency: Less overthinking, with 30% lower…pic.twitter.com/jFS7I40avsJune 12, 2026

Kimi K2.7 Codeは総パラメーター数1兆、アクティブパラメーター数320億のMoEモデルで、最大コンテキスト長は26万2144(256K)トークンです。

「Kimi K2.7 Code」「Kimi K2.6」「GPT-5.5(xhigh)」「Claude Opus 4.8(xhigh)」のベンチマーク結果が以下。Kimi K2.7 Codeは前世代モデルの** Kimi K2.6**と比べてコーディングタスクにおける性能が大きく向上しています。

AIエージェントは1つのタスクを実行するために無駄なトークン消費を積み重ねてしまうことがあります。Kimi K2.7 Codeは推論効率が向上しており、Kimi K2.6と比べて約30％少ない思考トークンでタスクを実行できます。以下のグラフは横軸が消費トークン数、縦軸が性能を示しており、Kimi K2.7 CodeがKimi K2.6と比べて高性能かつ高効率であることが分かります。
